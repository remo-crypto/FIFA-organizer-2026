# Security, Privacy & AI Governance Framework

This document specifies the security controls, privacy compliance measures, and AI safety guardrails for the **FIFA World Cup 2026 Smart Stadium Operations Platform**.

---

## 1. End-to-End Security Architecture

To protect physical assets, critical infrastructure, and fan telemetry, the platform implements a zero-trust architecture.

```
                  +--------------------------+
                  |    TLS 1.3 Encryption    |
                  +-------------+------------+
                                |
                                v
                  +--------------------------+
                  |   Envoy API Gateway      | <---> [ OIDC / OAuth2 Keycloak ]
                  |   - JWT Scope Checks     |
                  |   - WAF & Rate Limiting  |
                  +-------------+------------+
                                |
                                v
                  +--------------------------+
                  |  Encrypted VPC Mesh      |
                  |  - mTLS between Pods     |
                  +--------------------------+
```

### 1.1. Network Security & Data Encryption
* **In-Transit**: All communications are encrypted using TLS 1.3 for HTTPS and Secure WebSockets (WSS). Weak cipher suites are disabled at the Load Balancer level.
* **At-Rest**: Database disks are encrypted using AES-256 with keys managed in cloud Key Management Services (AWS KMS / GCP Cloud KMS).
* **Inter-Service Communication**: Microservices communicate inside a private VPC using Istio Service Mesh, enforcing mutual TLS (mTLS) for authorization between pods.

### 1.2. Identity & Access Management (IAM)
* Identity Provider is Keycloak (OpenID Connect / OAuth2 compatible).
* All API endpoints demand JWT verification. Roles are mapped to scopes:
  - `stadium:ops:write` (Operators)
  - `stadium:security:dispatch` (Security Teams)
  - `stadium:transit:write` (Transit Coordinators)
  - `stadium:fan:read` (Fans)

### 1.3. Threat Mitigation
* **Web Application Firewall (WAF)**: Configured at the edge (Cloudflare) to block SQL injections, cross-site scripting (XSS), and rate-limit excessive requests (e.g., maximum 60 requests/minute per fan client).
* **SQL Injection & ORM**: FastAPI models leverage SQLAlchemy/PG8000 parameterization to prevent injection vectors.

## 2. Global Privacy Compliance Framework

Since the tournament is held in the USA, Canada, and Mexico, the platform complies with multiple privacy legislations:
1. **USA**: CCPA / CPRA (California) and state-level consumer privacy laws.
2. **Canada**: Personal Information Protection and Electronic Documents Act (PIPEDA).
3. **Mexico**: Ley Federal de Protección de Datos Personales en Posesión de los Particulares (LFPDPPP).

### 2.1. Privacy by Design Implementations
* **Anonymization of CCTV Feeds**: Bounding box counts generated by YOLO edge models only extract integer statistics (`people_count`). Frames are immediately discarded at the edge, and zero facial templates or biometric metadata are ever transmitted to the cloud.
* **Data Minimization**: Fan location tracking via GPS/BLE is strictly opt-in. Data is stored transiently in Redis (TTL under 30 minutes) and deleted immediately after the match concludes.
* **Right to be Forgotten**: The Fan App includes a "Delete Account" button, which automatically triggers a transaction deleting user profiles from PostgreSQL and database backups.

## 3. AI Governance & Safety Guardrails

To prevent hallucinations, biases, and unsafe operations, a structured governance pipeline is enforced.

### 3.1. Hallucination Control in RAG
* **Strict Context Window**: LLM prompts restrict the AI from generating answers beyond the retrieved document segments.
  - *Prompt Template Constraint*: `"Answer the user question using ONLY the context provided below. If the context does not contain the answer, reply: 'I cannot verify this information based on official SOPs.'"`
* **Mandatory Citations**: The Operations Copilot must supply the source file name, section, and page number for every advice generated, allowing operators to verify the details.

### 3.2. Human-in-the-Loop (HITL) Validation
* **Recommendation Only**: The AI Operations Copilot is forbidden from executing direct changes to physical signage, dispatching emergency responders, or announcing stadium evacuations.
* **Audit Trail**: Every recommendation accepted or rejected by an operator is logged in the `audits` table in PostgreSQL, capturing the query, the AI output, the operator's response, and the timestamp.

### 3.3. Bias & Fairness Mitigations
* **Queue Distribution**: Forecasting models are audited weekly for systemic bias (e.g., verifying that wait-time predictions are equally accurate across all ticket tiers and entry gate zones).
* **Accessibility Inclusivity**: Generative voice outputs (Screen readers / Audio announcements) use standard text-to-speech engines calibrated to adjust playback speed and tone for visually and hearing-impaired users.
